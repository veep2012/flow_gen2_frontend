# Technical Debt Register

Use this folder to track technical debt items deferred during development.

## File naming
- One file per date: `YYYY-MM-DD.md`
- Example: `2026-03-05.md`

## Entry expectations
- Include concrete code locations (file paths).
- Explain impact and reason for deferral.
- Describe a proposed fix and completion signal.
- Keep status explicit (`open`, `in_progress`, `resolved`).

## Ownership
- Debt entries may be created by agents when debt is identified and cannot be fixed in the current task scope.
