## TD-20260319-05
- Status: resolved
- Title: Revision-code bootstrap and seed path needs explicit identity and foreign-key safety validation
- Context: Review of the revision-code change noted that this PR evolves a reference-style table into a lifecycle-aware table. Under the current repository policy, database changes are handled by dropping and recreating the database from `ci/init/`, so bootstrap and seed behavior are the primary risk areas, especially when downstream tables reference `rev_code_id`.
- Location:
  - `ci/init/`
  - `tests/`
  - `documentation/`
- Impact/Risk: Even if the target schema is correct, a weak bootstrap path can break recreated environments through nondeterministic seed order, invalid FK transitions during inserts/updates, accidental identity remapping, or assumptions about idempotency that do not hold in repeatable setup flows. The highest-risk failure is changing `rev_code_id` identity semantics in a way that breaks downstream references without immediate visibility.
- Why Deferred Now: The current task scope was review and debt capture, not a full audit of bootstrap/seed behavior and downstream reference safety. Closing this properly required checking init/seed scripts, downstream references, and targeted compatibility tests.
- Proposed Fix: Add bootstrap notes and automated validation for:
  - deterministic seed order
  - foreign-key validity across insert/update phases
  - idempotency where repeatable setup flows expect it
  - preservation of published `rev_code_id` identity semantics when referenced externally
  If external tables refer to revision codes, add an explicit bootstrap note or automated test proving those references remain intact after recreation and reseeding.
- Acceptance Signal: Bootstrap and seed tests show that recreated environments initialize cleanly, `rev_code_id` references remain valid, and repository documentation states the expected identity-preservation and idempotency guarantees.
- Resolution:
  - Added automated bootstrap validation in `tests/api/api/test_revision_code_seed_contract.py` covering clean reinitialization via `ci/init/flow_init.psql` + `ci/init/flow_seed.sql`, stable seeded `rev_code_id` identities, downstream foreign-key safety, and post-seed sequence advancement.
  - Documented the supported bootstrap-only contract and identity-preservation guarantees in `documentation/api_db_rules.md` and `documentation/document_flow.md`.
  - Added scenario-first traceability for the bootstrap validation in `documentation/test_scenarios/revision_code_seed_contract_api_test_scenarios.md` and `tests/api/api/test_scenario_traceability.py`.
